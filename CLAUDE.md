## Architecture

**TypeScript ESM CLI** that reads Excel files via SheetJS and outputs RFC 4180 CSV to stdout.

### Default command routing
`src/index.ts` patches `process.argv` to insert `cat` when the first arg isn't a known subcommand. This makes `excel-cli file.xlsx` equivalent to `excel-cli cat file.xlsx`. Commander's `--no-header` flag inverts to `opts['header'] === false`.

### Data flow
`bin/excel-cli.js` → `dist/index.js` → Commander routes to command handler → `parser.ts` reads file into internal `Workbook` model → command applies filters → writes CSV to stdout.

### Key modules
- **`src/reader/parser.ts`** — SheetJS wrapper. `parseFile()` reads buffer into internal `Workbook`/`Sheet`/`Row`/`Cell` model. `resolveSheet()` resolves by name (case-insensitive) or 0-based index. Supports stdin via `-` filename.
- **`src/reader/types.ts`** — `Workbook`, `Sheet`, `Row`, `Cell`, `CellType` interfaces.
- **`src/commands/cat.ts`** — CSV output. Uses `resolveColumns()` and `applyRowSelection()` for filtering.
- **`src/utils/columns.ts`** — Column selector resolution. Priority order: header name match (case-insensitive) → Excel letter (≤3 chars) → 1-based numeric index. This ordering is critical — header names must be checked before Excel letters to avoid "Name" being parsed as column N-A-M-E.
- **`src/utils/csv.ts`** — Lightweight RFC 4180 CSV serializer (replaced `fast-csv` due to CJS/ESM incompatibility with tsup).
- **`src/utils/rows.ts`** — `RowRange` stores 1-based inclusive values. `applyRowSelection()` converts to 0-based internally.

### Conventions
- All data output goes to `process.stdout`, all errors/warnings to `process.stderr`.
- Exit codes: `0` success, `1` file/parse error, `2` invalid argument. Use `die(message, code)` from `utils/errors.ts`.
- CSV uses `\r\n` line endings per RFC 4180.
- Test fixtures in `test/fixtures/` are generated by `scripts/create-fixtures.ts` (run via `npx tsx scripts/create-fixtures.ts`).
- Tests capture stdout by monkey-patching `process.stdout.write`.

### Runtime dependencies
Only two: `xlsx` (SheetJS) for parsing and `commander` for CLI.
